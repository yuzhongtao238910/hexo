关于js之中的数据类型的转换的原则

原始 -》 数字 调用Number()
  • true 1
  • false 0
  • null 0
  • undefined NaN
  • NaN NaN
  • string
    • 空字符串（含空白字符\t\r\n）：0
    • 去掉引号：不是数字就是NaN
    • 存在空格的话，会去掉首尾的空格来转换
所有 -》 bool 调用Boolean()
  • 0 false
  • undefined false
  • null false
  • NaN false
  • 空字符串 false 只有空字符串是false
  • 其他都是true
原始 -》 字符串 调用String()
  • null “null”
  • NaN “NaN “
  • undefined “undefined”
  • number “数字”
  • true “true”
  • false “false”

对象 -》 原始

  • 调用valueOf 得到的是对象的话，就会调用toString()，如果toString得到的还是对象的话，就会报错

几个知识点

三个知识点：options预检请求，更换镜像地址，修改git-log

  • options预检请求

    什么时候会发送预检请求？一句话：在跨域&非简单请求的条件下会发送预检请求

    • 首先由于浏览器的同源策略，在不跨域的情况下是不会发送options请求的
    • 简单请求和非简单请求：可以理解为到底是否要对后台的数据做操作，最简单的get请求是通常都是单纯的读取数据，不涉及任何操作，所以是不会发送options请求
    • 简单请求需要满足以下的条件
      • 只使用以下HTTP方法之一：GET、HEAD或POST。
      • 只使用以下HTTP头部：Accept、Accept-Language、Content-Language、Content-Type。
      • Content-Type的值仅限于：application/x-www-form-urlencoded、multipart/form-data或text/plain。
