3- 浮动产生的原因

a) 原因：在文档流中，父元素的高度默认是被子元素撑开的，即子元素有多高，父元素就有多高。但是，当为子元素设置浮动后，子元素会脱离文档流，导致它们不再占据父元素中的空间，进而无法撑起父元素的高度，使得父元素的高度塌陷。
b）后果：父元素高度塌陷后，会导致父元素下的所有元素都会向上移动，这样会导致页面布局混乱。因为浮动元素已经脱离了文档流，所以它们不会受到父元素高度的限制，可能会与其他元素重叠或覆盖。

总结

  • 浮动会使元素脱离文档流。
  • 静态定位和相对定位不会使元素脱离文档流。
  • 绝对定位、固定定位和粘滞定位（在达到阈值后）会使元素脱离文档流。

关于一些很琐碎的知识点

一些很琐碎的知识点，晚上回想了一下，答错了好多，但这些实在是太细了，一不注意，就很容易忘记，还是那句话，没见过就不好回答啊

知识点1

  • 关于在vue的template之中使用v-if的效果 在最顶层的template使用的v-if是没有效果的，内部的template是可以使用的

知识点2

几种定位的方式

  • relative 相对定位
  • absolute 绝对定位
  • static 正常的布局
  • fixed 固定定位
  • sticky 粘性定位

知识点3

  • space-between: 在每行上均匀分配弹性元素。相邻元素间距离相同。每行第一个元素与行首对齐，每行最后一个元素与行尾对齐。

  • space-around: 在每行上均匀分配弹性元素。相邻元素间距离相同。每行第一个元素到行首的距离和每行最后一个元素到行尾的距离将会是相邻元素之间距离的一半。

  • space-evenly: flex 项都沿着主轴均匀分布在指定的对齐容器中。相邻 flex 项之间的间距，主轴起始位置到第一个 flex 项的间距，主轴结束位置到最后一个 flex 项的间距，都完全一样。

知识点4

  • 昨晚想到的微前端的几种有意思的东西
    • 多端口的话就需要开启跨域支持啊
    • public-path.js文件需要放置在首位
    • 对于需要鉴权的页面，是采用多端口还是单端口使用子路径啊？一会自己试一下

知识点5

  • grid布局，最近发现grid布局问了好多次，grid是二维布局，基于行和列进行布局，而flex是一维布局，只会基于主轴啊

2024年06月 关于面试之中两个很简单的小点

ts的交叉类型/联合类型，css之中的display: none 以及 visibility：hidden

交叉类型：使用&符号作为交叉类型的分隔符，表示这个类型同时包含了这些类型的所有特性。

联合类型：使用|符号作为联合类型的分隔符。可以是子类型之中的任意一种类型

display: none 不显示，并且不占用文档的内容区域

visibility：hidden 不显示，但是保留在文档之中的布局
